# 역할 대체 착각

## 핵심 주장

AI를 활용하면 다른 역할이 쉬워 보일 수 있다.

하지만 이것은 그 역할의 하방이 올라간 것이지, 전문 역할의 상방과 책임이 사라진 것이 아니다.

간단한 작업을 성공하다 보면 "이제 저 역할은 없어도 되겠다"는 생각이 들 수 있다.
하지만 더 복잡하고, 수익을 내야 하고, 지속적으로 유지되어야 하는 단계에 가면 그 생각이 틀렸다는 것을 알게 된다.

## 왜 착각이 생기는가

사람은 다른 역할을 자신이 이해할 수 있는 범위까지만 본다.

AI는 그 범위 안에서 꽤 그럴듯한 산출물을 만들어준다.

그래서 내가 아는 만큼의 일은 AI로 할 수 있어 보이고, 그 결과 역할 전체가 쉬워 보인다.

```text
내가 이해한 표면적 요구
  ↓
AI가 그럴듯한 산출물 생성
  ↓
간단한 성공 경험 축적
  ↓
다른 역할이 없어도 될 것 같은 착각
```

AI가 설명을 잘해주는 것도 이 착각을 강화한다.

다른 지식을 예전보다 훨씬 쉽게 습득할 수 있기 때문에, 그 분야를 더 잘 이해한 것처럼 느끼게 된다.

하지만 쉬운 입문과 전문적인 책임은 다르다.

## 역할별 착각 예시

```text
기획자
  -> AI로 개발 산출물이 나오니 개발자가 필요 없다고 느낄 수 있음

개발자
  -> AI로 화면/문서/기획 초안이 나오니 디자이너나 기획자가 필요 없다고 느낄 수 있음

디자이너
  -> AI로 구현 초안이 나오니 개발자가 필요 없다고 느낄 수 있음

프론트엔드 / 앱 개발자
  -> AI로 API나 서버 코드를 만들 수 있으니 백엔드 개발자가 필요 없다고 느낄 수 있음

백엔드 개발자
  -> AI로 UI 코드를 만들 수 있으니 프론트엔드 개발자가 필요 없다고 느낄 수 있음
```

이 착각은 특정 직군만의 문제가 아니다.

AI를 쓰는 누구나 겪을 수 있는 공통적인 인지 효과다.

## 하방 상승과 역할 소멸은 다르다

AI는 다른 역할의 기본적인 산출물을 만들 수 있게 해준다.

이것은 하방 상승이다.

하지만 하방 상승은 역할 소멸과 다르다.

```text
하방 상승
  = 입문 장벽이 낮아짐
  = 표면적 산출물을 만들 수 있음
  = 간단한 작업을 혼자 처리 가능

역할 소멸
  = 복잡한 판단과 책임까지 필요 없어짐
  = 품질, 유지보수, 운영 리스크까지 대체됨
  = 전문적 상방이 사라짐
```

AI가 올려주는 것은 주로 첫 번째다.

두 번째는 훨씬 높은 기준이다.

## 헛간과 빌딩

역할 대체 착각은 헛간과 빌딩 비유로 설명할 수 있다.

유튜브를 보고 내가 쓸 헛간을 만들 수는 있다.
주변에 있는 이것저것을 가져다 붙이고, 문이 잘 안 닫히면 판자로 막고, 물이 조금 새도 혼자 쓰는 창고라면 고쳐가며 쓸 수 있다.

하지만 수천 명이 생활하는 빌딩을 그렇게 만들 수는 없다.

빌딩은 내가 아는 만큼만 대충 만들 수 있는 대상이 아니다.
구조, 안전, 인프라, 운영, 유지보수, 보안, 비상 상황까지 여러 전문 역할의 판단이 필요하다.

```text
헛간
  = 내가 이해한 범위 안에서 직접 만들 수 있음
  = 유튜브와 임시방편으로 어느 정도 가능
  = 실패 비용이 낮음
  = 불편하면 고쳐가며 쓸 수 있음

빌딩
  = 여러 전문 역할의 깊은 판단이 필요함
  = 실패 비용이 큼
  = 구조, 안전, 운영, 유지보수 책임이 있음
```

AI가 어떤 역할의 헛간 수준 산출물을 만들 수 있게 해준다고 해서, 그 역할이 빌딩 수준에서 필요 없어지는 것은 아니다.

## 한 문장 요약

> AI는 다른 역할의 표면적 산출물을 만들 수 있게 해주지만, 그것은 하방 상승이지 전문 역할의 상방과 책임이 사라졌다는 뜻은 아니다.
